Bienvenido a la información sobre Sistemas de Información

1. La Información en la Empresa


La información es un recurso estratégico para la toma de decisiones dentro de una empresa. Su correcta gestión permite mejorar la eficiencia, competitividad y adaptación al entorno. Las empresas dependen de la información para coordinar operaciones, analizar tendencias, optimizar procesos y mejorar la experiencia del cliente.

2. Sistemas de información

Un sistema de información (SI) es un conjunto de componentes interrelacionados que recopilan, procesan, almacenan y distribuyen información para apoyar la toma de decisiones, la coordinación, el control y el análisis en una organización.

3. Tipos de Sistemas de Información

Computación en la nube


La computación en la nube (Cloud Computing) es un modelo que permite el acceso a recursos informáticos (servidores, almacenamiento, bases de datos, redes, software, etc.) a través de internet, sin necesidad de poseer una infraestructura física propia.

Características principales:

Acceso bajo demanda: Los recursos se pueden utilizar cuando se necesiten.

Escalabilidad: Permite aumentar o disminuir recursos según la demanda.

Pago por uso: Se paga solo por los recursos utilizados.

Ubicación remota: No es necesario tener los servidores en la empresa.

Seguridad y respaldo: Los proveedores garantizan alta disponibilidad y protección de datos.

Tipos de nube:

Nube Pública: Infraestructura compartida y administrada por proveedores externos (AWS, Azure, Google Cloud).

Nube Privada: Infraestructura dedicada a una sola organización.

Nube Híbrida: Combinación de nube pública y privada.

Modelos de negocio en la nube


Los modelos de negocio en la nube dependen de cómo se ofrecen los servicios y el valor que generan para las empresas. Se pueden clasificar en:

Software como Servicio (SaaS - Software como Servicio):

Aplicaciones accesibles a través de internet sin instalación.

Ejemplos: Gmail, Microsoft 365, Salesforce, Dropbox.

Modelo de negocio: Suscripción o pago por uso.

Plataforma como Servicio (PaaS - Plataforma como Servicio):

Entorno de desarrollo en la nube para crear aplicaciones sin gestionar la infraestructura.

Ejemplos: Google App Engine, Microsoft Azure App Services.

Modelo de negocio: Pago por uso o suscripción para desarrolladores.

Infraestructura como Servicio (IaaS - Infraestructura como Servicio):

Recursos como servidores, almacenamiento y redes proporcionadas por internet.

Ejemplos: Amazon EC2, Google Compute Engine, Microsoft Azure Virtual Machines.

Modelo de negocio: Pago por consumo de recursos.

Función como Servicio (FaaS - Función como Servicio):

Ejecución de código sin necesidad de gestionar servidores (arquitectura serverless).

Ejemplos: AWS Lambda, Google Cloud Functions.

Modelo de negocio: Pago por ejecución de funciones.
